## 一、项目背景

### 1.1 项目目标

构建完整的加密货币期权+永续合约量化交易数据采集系统，支持 6 类组合策略的回测与实盘运行：

| 策略 | 频率 | 核心数据依赖 | 优先级 |
|------|------|--------------|--------|
| Short Strangle + 永续对冲 | 日级 | 期权bid/ask、Delta、保证金率 | P0 |
| 合成备兑看涨 | 周级 | Call权利金、永续价格，资金费率 | P0 |
| 领口策略 | 月级 | Call/Put权利金差、Delta | P0 |
| 资金费率套利 + 期权保护 | 日级 | 资金费率、基差、Put权利金 | P1 |
| Gamma Scalping | 日级 | 实时Greeks、永续bid/ask | P1 |
| 波动率期限结构套利 | 周级 | 近远端IV，两组Greeks | P2 |

### 1.2 数据源

| 交易所 | 市场类型 | 数据覆盖 |
|--------|----------|----------|
| Deribit | 期权+永续 | BTC/ETH/SOL |
| Binance | 永续合约 | 全币种 |
| Hyperliquid | 永续合约 | 全币种 |

### 4.3 采集流程

```python
# 伪代码：采集流程
async def collection_loop():
    while running:
        # 1. 获取合约列表
        instruments = await fetch_instruments()

        # 2. 分批采集（50 个/批）
        for batch in batches(instruments, size=50):
            tickers = await batch_fetch_tickers(batch)
            books = await batch_fetch_orderbooks(batch)

        # 3. 写入存储
        save_to_parquet(tickers, books)

        # 4. 写入 SQLite
        save_to_sqlite(tickers, books)

        # 5. 睡眠 1 秒
        await sleep(1)
```

## 五、已识别风险与约束

### 5.1 高优先级风险

| 风险 | 描述 | 影响 | 缓解措施 |
|------|------|------|----------|
| R1 | WebSocket 频道超限（1678 合约 > 300 限制） | 无法订阅全量实时数据 | 分优先级订阅 |
| R2 | 数据间隙导致回测偏差 | 回测结果不准确 | 间隙检测 + 插值 |
| R3 | API 限流导致数据缺失 | 部分数据未采集 | 指数退避 + 重试 |
| R4 | 合约到期展期失败 | 实盘策略中断 | 自动展期机制 |

### 5.2 中优先级风险

| 风险 | 描述 | 影响 | 缓解措施 |
|------|------|------|----------|
| R5 | 存储空间不足 | 采集中断 | 保留策略 + 压缩 |
| R6 | 网络波动 | 数据不连续 | 断点续传 |
| R7 | 数据质量差 | 策略信号失真 | 异常值过滤 |
| R8 | 并发写入冲突 | 数据丢失 | WAL 模式 |

### 5.3 技术约束

| 约束 | 值 | 说明 |
|------|---|------|
| API 限速 | 20 req/s | Deribit REST |
| WebSocket 频道上限 | 300/连接 | 单连接最大订阅数 |
| Python 版本 | 3.11+ | 项目要求 |
| 存储保留 | 90 天 | Parquet 压缩 |
| 测试覆盖率 | >=90% | 核心路径覆盖 |

## 十八、关键性能指标（KPI）详细定义

### 18.1 数据质量 KPI

| 指标 | 定义 | 目标值 | 测量方法 | 告警阈值 |
|------|------|--------|----------|----------|
| **数据延迟** | 从 Deribit 到存储的时间差 | < 1s | `timestamp_storage - timestamp_api` | > 5s |
| **数据丢失率** | 预期 vs 实际记录数 | < 0.1% | `(expected - actual) / expected * 100` | > 1% |
| **数据重复率** | 重复记录数 / 总记录数 | < 0.01% | `duplicates / total * 100` | > 0.1% |
| **数据准确率** | 校验通过率 | > 99.9% | `valid_records / total * 100` | < 99.5% |
| **期权链完整率** | 完整合约数 / 总合约数 | > 95% | `complete_contracts / total_contracts * 100` | < 90% |
| **Greeks 有效率** | 有效 Greeks 记录 / 总记录 | > 95% | `valid_greeks / total_records * 100` | < 70% |
| **OI 零值占比** | OI=0 记录比例 | < 1% | `zero_oi_records / total * 100` | > 5% |

### 18.2 系统性能 KPI

| 指标 | 定义 | 目标值 | 测量方法 | 告警阈值 |
|------|------|--------|----------|----------|
| **吞吐量** | 每秒处理记录数 | > 500 条/s | Prometheus `rate(records_processed_total)` | < 200 |
| **采集延迟** | 单次采集循环耗时 | < 1s | Prometheus `collection_lag_seconds` | > 2s |
| **存储写入延迟** | 写入 SQLite/PQ 时间 | < 100ms | Prometheus `write_latency_seconds` | > 200ms |
| **API 错误率** | 4xx/5xx 响应占比 | < 1% | Prometheus `api_errors_total / api_requests_total` | > 2% |
| **WebSocket 重连次数** | 每小时重连次数 | < 10 次 | Prometheus `ws_reconnects_total` | > 30 |
| **内存使用率** | 进程内存占用 | < 80% | Prometheus `process_resident_memory_bytes` | > 90% |
| **CPU 使用率** | 平均 CPU 占用 | < 60% | Prometheus `process_cpu_seconds_total` | > 80% |
